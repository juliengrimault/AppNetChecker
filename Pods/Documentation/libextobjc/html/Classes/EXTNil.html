Overview

Like \c NSNull, this class provides a singleton object that can be used to represent a \c NULL or \c nil value. Unlike \c NSNull, this object behaves more similarly to a \c nil object, responding to messages with “zero” values. This eliminates the need for \c NSNull class or equality checks with collections that need to contain null values.

This class will pretend to be \c NSNull when queried for its class or compared for equality, to keep compatibility with code that expects or uses \c NSNull.

Note: Because this class does still behave like an object in some ways, it will respond to certain \c NSObject protocol methods where an actually \c nil object would not.
